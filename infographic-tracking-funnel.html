2 · Shortlink Architecture · How tracking works
Shortlink redirect = key ของระบบ

แทน QR ที่ชี้ตรงไป partner — ให้ QR ชี้มาที่ aqmx.co/r/xxx ก่อน → log → redirect ออก

1
User scans QR / sends SMS
aqmx.co/r/abc123
QR ที่จอ shows shortlink ของเรา ไม่ใช่ partner URL ตรงๆ
Logged QR/SMS event already captured at popup level
↓ HTTP request to our function
2
Firebase Function · /r/:id
GET https://aqmx.co/r/abc123
Lookup linkId → log event → 302 redirect
Logged to Firestore clickId, slotId, projectId, ts, ua, ip-hash, geo
↓ 302 redirect + append UTM
3
Partner site (booking page)
partner.com/book?room=A38&utm_source=mahogany-signage&utm_medium=qr&utm_campaign=spa-may26&utm_content=slot-7&clickId=abc123
Partner GA เห็น UTM · เรายังไม่เห็นอะไรเพิ่ม
Partner sees Traffic จาก mahogany-signage
Source/campaign/content ครบ
↓ (optional) Postback after conversion
4
Conversion postback (optional)
POST https://aqmx.co/api/conversion?clickId=abc123&value=2500&eventType=booking
Partner webhook → ระบบ match clickId → conversion event
We now know Conversion + revenue + ROI per slot

JS SDK

02
Webhook
Milestones ★
